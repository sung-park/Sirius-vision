============================================================
  Pet Vitality Insight - 웹앱 프로토타입
============================================================

반려동물의 센서 데이터와 보호자의 저널링 기록을 결합하여
'펫 활력 점수(Vitality Score)'를 시각화하는 모바일 웹 프로토타입.


------------------------------------------------------------
  실행 방법
------------------------------------------------------------

1. 의존성 설치 (최초 1회)

   npm install

2. 개발 서버 실행

   npm run dev

   → 브라우저에서 http://localhost:5173 으로 접속
   → 모바일 뷰를 위해 브라우저 개발자 도구(F12)에서
     모바일 디바이스 모드로 전환 권장 (iPhone 14 Pro 등)

3. 프로덕션 빌드

   npm run build

   → dist/ 폴더에 정적 파일 생성

4. 빌드 결과 미리보기

   npm run preview


------------------------------------------------------------
  테스트 시나리오
------------------------------------------------------------

[시나리오 B: 아픔 (기본값)]

  1. 앱 진입 → 활력 점수 65점 (노란색)
  2. "원인 파악을 위해 체크인해 주세요" 버튼 클릭
  3. 식사: 남김 → 배변: 설사 → 투약: 없음 → 특이사항: 평소와 다름 없음
  4. "저널 저장" 클릭
  5. 결과 확인:
     - 하트 플로팅 리워드 애니메이션
     - 점수 65 → 45점 (빨간색)으로 변경
     - 인사이트 카드 생성 (식이 알러지 의심, 컨디션 저조)

[시나리오 A: 건강함]

  1. 화면 하단 "시나리오 변경 (Demo)" 클릭
  2. "Scenario A: 건강함" 선택 → 점수 92점 (초록색)
  3. "모닝 체크인 작성하기" 클릭
  4. 식사: 완료 → 배변: 정상 → 투약: 없음 → 특이사항: 평소와 다름 없음
  5. 결과 확인:
     - 점수 92점 유지
     - 인사이트: "뽀삐의 컨디션이 좋습니다!"

------------------------------------------------------------
  주요 설계 결정
------------------------------------------------------------

- 다크 모드 기본: 데이터 가시성 확보를 위해 #0D0D1A 배경 사용
- Deep Purple 브랜드 컬러: Oura Ring의 신비로운 느낌 벤치마킹
- 모바일 퍼스트: max-width 480px, 데스크톱에서는 중앙 정렬
- 로컬 상태 관리: 프로토타입이므로 useState만 사용
- 목 데이터: 실제 센서 연동 없이 시나리오 기반 시뮬레이션
